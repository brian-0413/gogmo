# 機場接送派單平台 - 使用者體驗分析報告

## 研究範圍
分析司機端、派單方端、管理員端的關鍵使用者流程，識別痛點與改進機會。

---

## 一、主要用戶流程

### 司機端流程
1. **註冊** → 填寫個資/車輛/銀行 → 上傳行照+駕照+保險 → 等待審核
2. **接單大廳** → 瀏覽可用訂單 → 一鍵接單 → 前往執行行程
3. **小車頭** → 14步驟對話式發單引導 → 上架至接單牆
4. **帳務中心** → 查看交易紀錄/今日/本週/累積收入

### 派單方端流程
1. **註冊** → 填寫公司名稱/統編/電話 → 上傳證件 → 等待審核
2. **派單中心** → 選擇日期/車型 → 貼單解析 → 進入審核 → 逐筆編輯 → 發布
3. **行控中心** → 查看6格統計 → 管理訂單 → SSE即時更新狀態
4. **帳務中心** → 日期篩選 → 下載Excel → 標記已轉帳

### 管理員端流程
1. **費率設定** → 調整平台費率/退單費率/轉單費率 → 儲存
2. **帳號審核** → 切換司機/派單方Tab → 檢視文件與到期日 → 通過/拒絕

## 二、痛點分析

### 痛點一：銀行帳號驗證斷點（高優先）
**問題**：司機接單前需要銀行帳號，但這項資訊在註冊流程中只是「選填」，審核通過後才用橫幅提示填寫，造成使用者興致勃勃準備接單卻被中斷。

**改善建議**：將銀行帳號改為註冊Step 4的必要欄位，並在橫幅中提供直接跳轉連結。

### 痛點二：小車頭14步驟流程過長（中優先）
**問題**：對話式介面設計良好，但14個步驟違背了「小車頭」快速發單的初衷。包含行李尺寸/數量等瑣碎欄位，讓經驗豐富的司機感到繁瑣。

**改善建議**：
- 將行李流程縮短為「有/無」二元選擇
- 預設值：今天、乘客1人、無行李
- 資深司機可一鍵跳過非必要欄位

### 痛點三：派單方文件上傳門檻（高優先）
**問題**：「公司/商業登記核准公文」對小型車隊或個人派遣而言是沉重負擔。許多小型派單方可能連統編都沒有，卻被迫上傳看不懂的文件。

**改善建議**：
- 商業登記改為「選填」
- 提供替代方案：LINE群組截圖、司機名單、評價截圖等
- 審核時可依據實際需求個案評估

### 痛點四：批次發單失敗回饋不足（中優先）
**問題**：「解析失敗」或「建立失敗」時，只顯示模糊的錯誤訊息，使用者不知道如何修正。

**改善建議**：Parser API應回傳結構化錯誤，例如：缺少時間、格式無法解析、車型未選擇等。

### 痛點五：費率設定說明誤導（低優先）
**問題**：介面顯示「重啟伺服器後會重設為預設值」，但實際上可能已寫入資料庫，造成管理員困惑。

## 三、優先改進項目

| 優先順序 | 項目 | 原因 |
|---------|------|------|
| P0 | 銀行帳號改為必填 | 現有流程造成接單前斷點，直接影響營收 |
| P1 | 小車頭流程簡化 | 14步太冗長，違背工具初衷 |
| P1 | 商業登記改選填 | 阻擋小型派單方加入 |
| P2 | 批次發單錯誤訊息結構化 | 改善除錯體驗 |
| P3 | 費率設定提示更新 | 避免誤解 |

---

## 四、結論

整體而言，平台的使用者體驗已建立良好的基礎：
- 即時SSE推播提升互動感
- 6格統計提供一目了然的數據概覽
- 智慧排班功能具有獨特價值

最大的流失風險在於**新用戶的第一哩路**：從註冊到成功接單之間存在太多摩擦點（文件上傳、審核等待銀行帳號補填）。優先解決這些斷點，將能有效提升用戶留存率。
