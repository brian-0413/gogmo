# gogmo 派單分區與 Travel Time Matrix v1.0

> 本文件定義 gogmo 智慧排單系統使用的 11 個分區（zones），以及各 zone 之間的車程時間估算矩陣（travel time matrix）。
>
> **用途**：供智慧排單演算法判斷訂單距離相鄰程度、司機配套推薦、預估空車移動成本。

---

## 設計原則

1. **合併原則**：車程時間相近（差距 5 分鐘內）、地理相鄰、機場接送路徑類似的區域合併為同一 zone
2. **涵蓋範圍**：雙北（台北市 + 新北市）作為核心，機場與外圍區域獨立標記
3. **Matrix 基準**：平日非尖峰（上午 10:00 / 下午 2:00）的車程時間
4. **尖峰調整**：實際使用時，尖峰時段（7-9 早、17-20 晚）需乘以 1.3，深夜（23:00-5:00）乘以 0.8
5. **版本**：v1.0（2026-04）—— 初版基於地理估算，後續根據實際訂單數據校準

### 對稱性說明
此 matrix 採「對稱假設」—— 即 Z1 → Z6 的時間與 Z6 → Z1 相同。實際因為單行道、國道方向等因素可能略有差異，但誤差在 5 分鐘內，對派單決策影響小。

### 解讀範例
- Z1（大安區） → Z6（板橋）= 25 分鐘（非尖峰）
- Z6（板橋） → Z10（林口）= 25 分鐘（非尖峰）
- Z11（淡水） → Z9（新店）= 60 分鐘（跨南北，非尖峰最遠）

## Zone 相鄰判定

用於智慧排單的「距離分數」計算：

| 分類 | 車程時間 | 距離分數 | UI 標示 |
|---|---|---|---|
| **同區** | 10 分鐘內 | 100 | 💚 同區域 |
| **相鄰** | 10-20 分鐘 | 85 | 💚 相鄰區域 |
| **次近** | 20-35 分鐘 | 65 | 🟡 次近區域 |
| **較遠** | 35-50 分鐘 | 35 | 🟠 較遠區域 |
| **遠距** | > 50 分鐘 | 10 | 🔴 遠距區域 |

## 時段倍率調整

實際使用 matrix 時，需根據時段乘以倍率：

| 時段 | 倍率 | 說明 |
|---|---|---|
| 早尖峰 | ×1.3 | 07:00-09:00 平日 |
| 晚尖峰 | ×1.4 | 17:00-20:00 平日 |
| 一般時段 | ×1.0 | 09:00-17:00 / 20:00-23:00 |
| 深夜 | ×0.8 | 23:00-05:00 |
| 週末假日 | ×1.1 | 整日（觀光區去程 ×1.3） |

### 計算範例
- 週一 08:00 從 Z1 → Z6：25 分鐘 × 1.3 = **33 分鐘**
- 週六 15:00 從 Z11（淡水觀光）→ Z1：45 分鐘 × 1.1 × 1.3（觀光區）= **64 分鐘**

## 資料結構（程式碼參考）

### TypeScript 定義

```typescript
// lib/zones/types.ts

export type ZoneCode =
  | 'TPE_EAST' | 'TPE_WEST' | 'TPE_NORTH' | 'TPE_NORTHEAST' | 'TPE_SOUTH'
  | 'NTPE_WEST' | 'NTPE_NORTHWEST' | 'NTPE_NORTHEAST' | 'NTPE_EAST' 
  | 'NTPE_WESTEXT' | 'NTPE_COASTAL';

export type AirportCode = 'AIRPORT_TPE' | 'AIRPORT_TSA' | 'AIRPORT_RMQ' | 'AIRPORT_KHH';

export type ExternalZoneCode = 
  | 'TAOYUAN_CORE' | 'TAOYUAN_NORTH' | 'TAOYUAN_SOUTH' 
  | 'HSINCHU' | 'MIAOLI' | 'YILAN_NORTH' | 'YILAN_SOUTH';

export type AnyZone = ZoneCode | AirportCode | ExternalZoneCode;

export interface DistrictMapping {
  district: string;        // 例：「板橋區」
  city: string;            // 例：「新北市」
  zone: ZoneCode;          // 例：NTPE_WEST
}

export interface TravelTimeEntry {
  from: AnyZone;
  to: AnyZone;
  minutes: number;         // 平日非尖峰基準
}
```

### 查詢函式

```typescript
// lib/zones/travelTime.ts

import { TRAVEL_TIME_MATRIX } from './matrixData';

export function getTravelTime(
  from: AnyZone, 
  to: AnyZone, 
  options?: { 
    time?: Date;
    isWeekend?: boolean;
  }
): number {
  const base = TRAVEL_TIME_MATRIX[from]?.[to];
  if (base === undefined) return 60; // 未定義預設 60 分鐘保守值
  
  let multiplier = 1.0;
  
  if (options?.time) {
    const hour = options.time.getHours();
    if (hour >= 7 && hour < 9) multiplier *= 1.3;
    else if (hour >= 17 && hour < 20) multiplier *= 1.4;
    else if (hour >= 23 || hour < 5) multiplier *= 0.8;
  }
  
  if (options?.isWeekend) multiplier *= 1.1;
  
  return Math.round(base * multiplier);
}

export function getDistanceScore(travelTime: number): number {
  if (travelTime <= 10) return 100;
  if (travelTime <= 20) return 85;
  if (travelTime <= 35) return 65;
  if (travelTime <= 50) return 35;
  return 10;
}
```

## 未來校準計畫

這份 matrix v1.0 是**地理估算**，不是實際運營數據。上線後應收集：

1. **訂單實際完成時間**：司機接單 → 抵達終點 → 下一單起點的實際時間
2. **區域間實際車程**：從 Google Maps API 或司機 GPS 軌跡推算
3. **時段倍率校準**：比對「演算法預估」與「實際車程」，自動微調倍率

**建議**：上線 1 個月後，用實際數據重算 matrix，發布 v2.0。
