## 第一期：司機端完善（現在要做）

### 1.1 司機行程狀態更新（進行中）

**現況**：API 已有 ACCEPTED→ARRIVED→IN_PROGRESS→COMPLETED，但 UI 無按鈕。

**需求**：
- 手機介面：點擊卡片展開大畫面（佔螢幕大部分）
- 4鍵流程：**開始 → 抵達 → 客上 → 客下**
- 時間鎖定：
  - 行程時間前 3 小時：只有「退單」可點，其餘反白
  - 行程時間前 3 小時內：「退單」反白，4鍵可依序點選
- 派單方行控中心同步亮燈（標籤形式對應狀態）
- 「客下」後自動歸入帳務中心

**規格文件**：待建立 → `docs/driver-status-flow.md`

### 1.2 司機端已完成行程回溯區塊

**需求**：
- 「帳務中心」Tab 新增「已完成行程」區塊
- 每筆已完成行程顯示：日期、起訖點、金額、乘客
- 資料保留 6 個月
- 可依日期範圍篩選

**相依功能**：依賴 1.1 完成後的資料流

### 1.3 派單方行控中心燈號同步

**需求**：
- 司機端狀態（開始/抵達/客上/客下）即時同步到派單方
- 卡片上以標籤/燈號形式顯示（與司機端同步）
- 需實作 SSE 或 polling 讓派單方接收狀態更新

**相依功能**：依賴 1.1

## 第三期：通訊與運營

### 3.1 即時訊息系統

**需求**：
- 司機與派單方之間的即時訊息
- 可綁定 LINE 通知（參考 LINE Messaging API）
- 或平台內建即時訊息（WebSocket/SSE）

**備註**：可考慮串接 LINE 官方帳號作為通知管道

### 3.2 通知系統

**需求**：
- 司機：新訂單推播、轉單請求、小隊支援請求、派單方訊息
- 派單方：新司機加入、轉單請求確認、行程狀態變更、司機訊息
- 推送方式：in-app（已上線用戶）+ LINE 推送（可選）

### 3.3 司機端即時位置追蹤

**需求**：
- 司機上報 GPS 位置（定期輪詢或 Geolocation API）
- 派單方行控中心顯示司機即時位置
- 司機狀態連動（ONLINE→接單→BUSY→完成還原）

### 3.4 自動化結算

**需求**：
- 每日凌晨自動結算前一交易日
- 派單方收入/支出汇总
- 司機收入/平台費汇总
- 產出結算報表

## 第四期：平台治理

### 4.1 Admin 後台

**需求**：
- 用戶管理（司機、派單方審核）
- 訂單總覽與管理
- 平台營收統計
- 投訴處理

### 4.2 派單方點數帳戶

**需求**：
- 派單方也需要預充值點數
- 刪除司機已接的單時，扣派單方 5% 補貼司機
- 點數不足時無法發布新訂單

### 4.3 轉帳確認功能

**需求**：
- 司機可申請提領餘額
- Admin 審核後執行轉帳
- 轉帳記錄追蹤

## 開發原則

1. **每個功能走完流程**：brainstorm → 規格文件 → 實作 → commit → 更新 CURRENT_WORK.md
2. **規格文件領先實作**：不要邊做邊想，先把規格寫清楚
3. **CURRENT_WORK.md 是事實來源**：每次 commit 後更新並 push
4. **三模型協作**：Haiku 做探索/Sonnet 做中等複雜度/Opus 做複雜架構決策
5. **UI 文字繁體中文、commit message 中文不用 emoji**
