# 小車頭專區 — 司機自助發單設計規格

## 1. 功能定位

- **目標用戶**：Premium 等級司機
- **核心功能**：司機在後台自行發單上架至接單大廳，其他司機可看見並接單
- **訂單標籤**：顯示「司機自派」標籤
- **門檻控制**：普通司機可見 Tab 入口但發單按鈕鎖定（灰色）

## 2. 呈現方式

- **Tab 分頁**：在司機後台新增「小車頭」分頁，與「接單大廳」「我的行程」「帳務中心」並列
- **介面風格**：LINE 風格對話泡泡，逐步引導式填單（非傳統表單）

## 3. 對話流程（共 12 步）

### Step 1 — 行程類型
- 問題：「您好！請問是接機/船還是送機/船？」
- 選項：接機/船、送機/船（二選一按鈕）

### Step 2 — 上車地點（自動更新類型）
- 問題：「好的！請選擇上車地點：」
- 選項（6格）：桃園機場、松山機場、清泉崗、小港、基隆港、其他
- **連動行為**：選擇後自動將 Step 1 的類型更新為：
  - 桃園/松山/清泉崗/小港 + 接 → 「接機」
  - 桃園/松山/清泉崗/小港 + 送 → 「送機」
  - 基隆港 + 接 → 「接船」
  - 基隆港 + 送 → 「送船」
  - 其他 → 維持接機/送機

### Step 3 — 航班日期時間
- 問題：「已設定為【接機】！請填寫航班日期和時間：」
- 輸入：date + time（橫排）
- **航班必填邏輯**：
  - 類型為接機/接船 → 航班必填
  - 類型為送機/送船 → 航班選填

### Step 4 — 航班號碼
- 問題：「請輸入航班號碼（必填）：」（或「（選填）」）
- 輸入：文字框（如 BR 32）+ 下一步按鈕

### Step 5 — 車型
- 問題：「請選擇車型：」
- 選項（3格）：小車(5人)、休旅(7人)、9人座

### Step 6 — 乘客數
- 問題：「請問乘客數？」
- 選項（4x2 網格）：1人、2人、3人、4人、5人、6人、7人、8人

### Step 7 — 行李（Loop 設計）
- **Loop A — 尺寸選擇**：
  - 問題：「請問有什麼行李尺寸？」
  - 選項（3x2 網格）：胖胖箱、28吋、24吋、20吋、其他、無行李
  - 選「無行李」→ 直接結束行李流程
- **Loop B — 數量選擇**（選擇特定尺寸後出現）：
  - 問題：「選擇了【24吋】，請問有幾件？」
  - 選項：1件、2件、3件（3格）
- **Loop C — 確認/繼續**（選擇數量後出現）：
  - 問題：「已加入【24吋 x 1】」
  - 顯示已選行李晶片（含移除按鈕 x）
  - 選項（2格）：
    - 「確定，沒了」→ 結束行李流程，進入下一步
    - 「還有其他尺寸」→ 返回 Loop A

### Step 8 — 上/下地點
- **接送機/船**：依類型自動設定一端
  - 接機/接船：上車地點自動填入選擇的機場/港口，下一步問目的地
  - 送機/送船：目的地自動填入選擇的機場/港口，下一步問上車地點
- 問題：「已設定【桃園機場 → 接機】請填寫目的地：」（或「請填寫上車地點：」）
- 輸入：文字框（如新竹火車站）+ 下一步按鈕

### Step 9 — 聯絡人
- 問題：「請輸入聯絡人和電話：」
- 輸入：姓名文字框 + 電話文字框（橫排）

### Step 10 — 費用模式
- 問題：「請選擇費用模式：」
- 選項（2格卡片）：
  - **客下轉帳**：派單人已收錢，直接匯給司機
  - **代收現金**：司機收現後，回金傭金給派單人
- 選擇「客下轉帳」→ 直接進入 Step 11
- 選擇「代收現金」→ 展開金額輸入（見 Step 11 代收分支）

### Step 11 — 金額

**客下轉帳模式**：
- 問題：「請輸入司機實拿金額（派單人匯給司機的金額）」
- 輸入：NT$ 文字框

**代收現金模式**：
- 問題：「選擇了代收現金模式，請填寫：」
- 輸入：
  - 代收金額（NT$）
  - 回金給派單（NT$）
- **即時計算**：司機實拿 = 代收金額 - 回金給派單，隨填寫即時更新顯示

### Step 12 — 特殊需求（可複選）
- 問題：「請選擇特殊需求（可複選）：」
- 選項（3格）：舉牌、安全座椅、其他

### Step 13 — 摘要確認
- 顯示完整發單摘要：
  - 類型、航班、時間
  - 上車、目的地
  - 車型、乘客/行李
  - 費用模式
  - 代收金額、回金金額、司機實拿（代收模式）
  - 特殊需求
- 按鈕：
  - 「確認發單上架」（主要按鈕，粉紅色）
  - 「我要修改」（次要按鈕，白底紅框）→ 點擊後回到 Step 1 重新填寫

## 4. 資料庫變更

### Prisma Schema
在 `Order` model 新增欄位：
```prisma
isSelfPublish  Boolean  @default(false)
```

## 5. API 設計

### POST /api/orders/self-publish
司機自助發單，建立 PUBLISHED 狀態訂單。

**Request Body**：
```json
{
  "orderType": "pickup" | "dropoff" | "pickup_boat" | "dropoff_boat",
  "scheduledTime": "2026-04-08T15:30:00.000Z",
  "flightNumber": "BR 32",
  "vehicleType": "small" | "suv" | "van9",
  "passengerCount": 2,
  "luggage": [
    { "size": "24吋", "quantity": 1 }
  ],
  "pickupLocation": "桃園機場",
  "dropoffLocation": "新竹火車站",
  "contactName": "王小明",
  "contactPhone": "0912-345-678",
  "feeMode": "transfer" | "cash_collection",
  "driverAmount": 2000,
  "cashCollected": 2200,
  "commissionReturn": 200,
  "specialNeeds": ["舉牌"],
  "notes": ""
}
```

**Response**：`{ success: true, data: Order }`

**授權**：需為 Premium 司機，否則回傳 403。

## 6. Premium 門檻控制

- 司機登入後，API `/api/auth/me` 回傳用戶資訊需含 `isPremium` 欄位
- 前端「小車頭」Tab：
  - Premium 司機：完整對話流程，Step 13 顯示「確認發單上架」可點擊
  - 普通司機：Tab 可見但按鈕鎖定，顯示提示「Premium 功能，請聯繫客服升級」
