## 邊界情況

### EDGE-01: 沒有日期（當天的單）
輸入沒有日期時，date 設為 null，由前端預設為當天。

### EDGE-02: 時間格式混用
- 「23:10」和「2310」都要能解析為 23:10
- 「06:00」和「0600」都要能解析為 06:00
- 「0045」要解析為 00:45（凌晨航班很常見）
- 「0:10」要解析為 00:10（注意只有一位數小時）

### EDGE-03: 車型在獨立行
某些派單方會先寫一行「小車」或「大車」，下面列出多筆訂單，代表這些訂單都是這個車型。解析時要能處理這種「上下文繼承」。

### EDGE-04: emoji 符號
有些派單方喜歡用 emoji 數字（如 (one)(two)(three)）當編號，需要轉換成阿拉伯數字。常見的 emoji 對照：(one)=1, (two)=2 ... (nine)=9, (zero)=0。

### EDGE-05: 金額前綴
金額前可能有 $、($)、💲 或什麼都不帶，都要能正確提取數字。

### EDGE-06: UI 預設值 + 行內金額覆蓋（核心流程）

派單方在 UI 上選好：
- 日期：3/30
- 種類：接機（注意：整批統一，但下面有「送」的行，種類仍由 UI 決定，備註會保留原文讓派單方在預覽時修正）
- 車型：任意R牌
- 車牌：R
- 預設金額：700

**貼入文字框：**
```
0:10接土城
0:35接大安+增高 $800
0:35接大安
0:55接淡水沙崙路 $800
1:00接萬華
1:30接板橋
```

**期望：解析為 6 筆訂單，全部繼承 UI 預設值。**

| # | 時間 | 種類 | 車型 | 金額 | priceSource | 備註 |
|---|------|------|------|------|-------------|------|
| 1 | 00:10 | 接機 | 任意R牌 | 700 | default | 接土城 |
| 2 | 00:35 | 接機 | 任意R牌 | 800 | inline | 接大安+增高 |
| 3 | 00:35 | 接機 | 任意R牌 | 700 | default | 接大安 |
| 4 | 00:55 | 接機 | 任意R牌 | 800 | inline | 接淡水沙崙路 |
| 5 | 01:00 | 接機 | 任意R牌 | 700 | default | 接萬華 |
| 6 | 01:30 | 接機 | 任意R牌 | 700 | default | 接板橋 |

### EDGE-07: 沒有預設金額，每筆自帶

派單方 UI 預設金額留空白。

**貼入文字框：**
```
0:10接土城 $700
0:35接大安 $800
1:00接萬華
```

**期望：**
- 前兩筆正常解析
- 第三筆「1:00接萬華」→ ⚠️ 警告「此筆未設定金額，請手動補填」

### EDGE-08: 相對日期
「明天」「後天」「今天」如果出現在文字框內（而非 UI 選擇），忽略不處理，因為日期已由 UI 確定。文字框內的日期相關文字當作備註保留。
