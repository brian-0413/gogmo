# gogmo 標準派單格式 v1.2

> 這份文件是給**派單方**看的派單規範。  
> AI 解析引擎依照此規範運作 —— 符合規範的訊息會被快速派出，不符合的會被退回修正。

---

## 為什麼要用 gogmo（不是 LINE）

LINE 群派單的世界裡，你需要不斷重貼訊息怕被洗掉、寫「一套不拆」怕司機只接一趟、用各種縮寫趕著貼單、用 emoji 與 ★ 裝飾求被看到。**這些行為是 LINE 的限制造成的，不是派單該有的樣子**。

gogmo 提供的東西，LINE 永遠做不到：

| 你需要的能力 | LINE 群 | gogmo |
|---|---|---|
| 訂單可視化（卡片、狀態、時間軸） | ❌ | ✅ |
| 即時掌握司機進度（接單、出發、抵達） | ❌ | ✅ |
| 預先審查司機三證（駕照、行照、營業證） | ❌ | ✅ |
| 帳務中心（自動結算、報表、發票） | ❌ | ✅ |
| 訊息中心（與司機 1 對 1 溝通、糾紛紀錄） | ❌ | ✅ |
| 訂單搜尋（找半年前那單跑哪去） | ❌ | ✅ |
| 重複貼單自動合併 | ❌ | ✅ |
| 司機評價系統（誰跑得好誰再合作） | ❌ | ✅ |

**只要你開始用 gogmo，效率提升是必然的**。我們不是「另一個 LINE 群」，而是**真正為派單而生的工具**。

也因為這份完整性，我們需要請你配合一些規範，讓系統能持續為所有派單方服務。

## 最小可接受格式

### 單筆訂單（最簡）

```
日期 時間 類型 地點 價格
```

✅ 範例：
```
4/19 22:10 接機 大安區信義路 850
4/20 07:00 送機 台中南屯 1500
```

### 完整版（推薦）

```
日期 時間 類型 地點 [車型] [航班號] 價格
```

✅ 範例：
```
4/19 22:10 接機 大安區信義路 小車 CI920 850
4/20 07:00 送機 台中南屯 大車 1500
```

## 五大必填欄位

| 欄位 | 規則 | ✅ 接受 | ❌ 不接受 |
|---|---|---|---|
| **日期** | 月/日 或 完整日期 | `4/19`、`2026/4/19` | `明天`、`後天`、`下週一` |
| **時間** | 24 小時制 HH:MM | `22:10`、`07:00`、`0:30` | `早上`、`晚上`、`七點半` |
| **類型** | 接機 / 送機 / 包車 / 接駁 | `接機`、`送機`、`接`、`送` | `R`、`配車`、空白 |
| **地點** | 區或更具體 | `大安區`、`大安區信義路` | `市區`、`隨便`、空白 |
| **價格** | 純數字（單位：新台幣） | `850`、`1500` | `便宜一點`、`看情況` |

> **接機/送機的隱含規則**：  
> 「接機」預設機場 = 桃園機場（TPE），終點 = 你寫的地點  
> 「送機」預設機場 = 桃園機場（TPE），起點 = 你寫的地點  
> 如果是松山（TSA）、高雄小港（KHH）、台中（RMQ），請明確標註：`4/19 22:10 接機(TSA) 大安區 850`

## 多筆訂單（同一則訊息）

每筆訂單**獨立一行**，用空行分隔：

✅ 正確：
```
4/19 22:10 接機 大安區信義路 850

4/19 22:30 接機 新店安民街 900

4/19 22:35 接機 內湖行善路 900
```

❌ 錯誤（多筆塞一行）：
```
4/19 22:10接大安22:30接新店22:35接內湖
```

## 平台不接受的情境

### 1. 套裝單（一套不拆）

**原因**：套裝單會被拆成獨立訂單處理。

派單方在 LINE 上寫「一套不拆」是因為 LINE 沒辦法把訂單綁在一起，只能用文字求司機一次接下整套。但這種綁定方式**對你跟司機其實都不利**：

- **對你不利**：你的人為套裝邏輯（例如送機 + 接機綁同一台車）通常不是最佳安排。gogmo 的智慧排程會找到更省成本、更高司機接單意願的組合
- **對司機不利**：被迫一次接 N 趟，沒辦法靈活配合自己當天的行程
- **對平台不利**：套裝單的解析、撮合、結帳邏輯複雜度遠高於獨立訂單，是少數但成本極高的特例

**處理方式**：  
平台會將你的套裝單**自動拆分為獨立訂單**，每筆獨立派發給司機，每筆獨立計價。系統會在 App 內通知你：

> ⚠️ 偵測到套裝單寫法（例：「一套不拆」「=1-2=」「+」連接多趟）  
> 已將這 N 筆訂單**拆分為獨立訂單**，請在派單列表中個別檢視。  
> 如果你希望司機一次接下整套，建議：  
> 1. 把總價平均分配到每筆訂單（讓單筆價格夠吸引人）  
> 2. 對其中任一筆使用「一鍵加價」功能，讓司機知道這趟特別需要人接

### 2. 範圍價格 / 模糊敘述

不接受 `1500-2000`、`看情況`、`便宜一點` 等寫法。價格必須是明確數字，司機才能即時決定是否接單。

### 3. 純廣告或非訂單訊息

`✈️包車找我❤️`、`好友已🈵`、`接單@後私訊` 等，會被自動過濾，不視為訂單。

## 帳號規則

### 實名註冊（強制）

註冊 gogmo 派單方帳號需提交：
- 真實姓名（與身分證一致）
- 統一編號（個人戶不需要）
- 行動電話（需簡訊驗證）
- 電子信箱
- LINE ID（可選，用於司機聯繫）

> **為什麼強制實名**：避免人頭帳號濫用平台、惡意貼單干擾系統。實名讓平台與司機都能信任你的訂單。

### 每日派單上限

| 等級 | 每日成功訂單上限 | 月費 |
|---|---|---|
| **基本派單方** | 100 單 / 日 | 免費 |
| **進階派單會員** | 500 單 / 日 | 🚧 即將推出 |
| **企業派單方** | 無上限 + 客製化整合 | 🚧 即將推出（議價） |

> **為什麼有上限**：  
> 每筆訂單的解析、撮合、訊息推播都需要平台運算資源。免費基本版的 100 單已經能滿足 95% 派單方的需求。  
> 如果你的業務量大於此，請升級進階派單會員 —— 這也讓我們能持續優化平台。

> **計算方式**：只計算「成功建立的訂單」。被系統 reject 的訂單**不計入**配額。

## 一鍵加價（提升訂單能見度）

### 如何使用
- 在派單列表的**任一張行程卡片右上角**，會顯示「💰 加價」按鈕
- 此按鈕**僅在訂單預定時間前 6 小時內**才會啟用（避免提早佔用「加價單」版面）
- 點擊後可選擇加價金額（建議 +50 / +100 / +200 / 自訂）
- 確認後，訂單即標記為「加價單」並進入司機端的「💰 加價單列表」優先排序

### 司機端的「加價單列表」
- 司機 App 有獨立的「加價單」分頁
- 加價單按「加價幅度」排序（加越多越前面）
- 司機可以優先挑這些單來跑
- 一般訂單仍正常顯示在主列表

### 加價防剝削機制（重要）

為了避免「先開低價搶置頂、再加價回正常價」的不健康競爭，平台規定：

1. **加價歷史完全公開**  
   每筆加價單會顯示完整加價歷史（時間 / 原價 / 現價）給司機看  
   例：`原價 600 → +50（4/19 16:00）→ +100（4/19 18:00）= 現價 750`

2. **加價次數上限**  
   每筆訂單**最多加價 3 次**

3. **價格下限保護**  
   訂單初始價格不得低於該區域 + 該時段的歷史均價 80%  
   - 例：大安區夜間接機歷史均價 800，則新訂單初始價不得低於 640
   - 系統會在派單方輸入價格時即時提示「您的價格低於該區域均價 X%，建議調整」
   - 派單方仍可堅持低價發單，但**該訂單的「加價」按鈕會被永久停用**（避免低價陷阱）

4. **加價單比例監控**  
   單一派單方的「加價單」比例若超過總訂單的 30%，系統會提示：「您的訂單加價比例偏高，可能初始定價過低，建議調整定價策略」

## 系統如何處理重複貼單

**你不需要重貼訂單**。在 LINE 群裡訊息會被洗掉，所以派單方習慣每隔幾分鐘重貼一次。

**在 gogmo 上**：
- 訂單卡會常駐在派單列表，不會被任何訊息洗掉
- 系統會自動偵測 30 分鐘內的相同訂單，**自動合併為一筆**
- 你看到的訂單卡只會有一張，無論你貼了幾次

**如果擔心訂單沒被司機看到**：
- 訂單卡上會顯示「目前 N 位司機已查看」
- 使用「一鍵加價」吸引司機接單（見上方）
- 不需要重貼

## ❌ 不接受的格式（會被退回）

| 違規類型 | 例子 | 為什麼不行 |
|---|---|---|
| 模糊時間 | `早上、晚上、七點多` | AI 無法判斷具體時間 |
| 相對日期 | `明天、後天、下週` | 會在跨日訊息中出錯 |
| 隱含地點 | `0:05三重送`（沒寫日期） | 缺日期無法排程 |
| 範圍價格 | `1500-2000`、`看情況` | 司機無法決定是否接 |
| 一行多筆 | `5:00信義/北投/南港 各900` | 解析容易出錯 |
| 純廣告 | `✈️包車找我❤️` | 不是訂單 |

## 違規處理機制

| 次數 | 處理 |
|---|---|
| 第 1 次（同類型違規） | AI 自動退回，附上「**改寫成標準格式的版本**」給你參考 |
| 第 2 次（同類型違規） | 系統提示 + 規範連結 |
| 第 3 次以上 | 該批次訂單需在 App 內手動補正欄位才能派出 |

> **AI 改寫範例**  
> 你貼：`0:05三重送`  
> AI 回：「⚠️ 這則訊息缺日期。下次請寫成：`4/19 00:05 送機 三重區 700` ← 點此複製」

> **設計理念**：我們不用懲罰機制管理派單方。違規越多次，AI 越會主動引導，但訂單仍會嘗試解析，你可以在 App 內手動補完缺失欄位。

## 信譽分數（🚧 即將推出）

每個派單方有 0-100 分的信譽分數，影響訂單**對司機的曝光優先順序**：

- 高分派單方：訂單**置頂顯示**給司機
- 中分派單方：正常排序
- 低分派單方：往下排

加分項：連續 7 天使用標準格式、司機接單後完成率高、特殊需求清楚明確  
扣分項：違規格式累積、訂單臨時取消、司機反映派單資訊不實、加價單比例異常

## FAQ

**Q: 我習慣用 LINE 貼，能不能照舊？**  
A: 短期內可以，AI 會盡量解析。但 AI 解析完會把訂單**改寫成標準格式**回給你看 —— 久了你就會習慣。重要的是，照標準格式貼，**訂單能更快派出去**。

**Q: 司機指定要求（無菸、會說台語等）怎麼寫？**  
A: 用括號標註在地點後面：`4/19 22:10 接機 大安區 (無菸司機) 850`。或在 App 上設定「司機偏好」🚧 即將推出。

**Q: 我有 100 筆訂單一起貼會不會被當廣告？**  
A: 不會。但建議用「批次匯入」🚧 即將推出 功能，可從 Excel 直接貼。

**Q: 預設價格能不能設定一次套用所有？**  
A: 目前每筆訂單的價格需獨立明寫。未來會推出「派單偏好」🚧 即將推出 功能。

**Q: 為什麼不對基本派單收費？**  
A: 平台初期希望降低派單方的使用門檻。未來會推出**進階派單會員**（每日 500 單、優先客服、批次匯入等加值功能），但基本 100 單派單持續免費。

**Q: 我超過 100 單怎麼辦？**  
A: 第 101 筆起會暫時無法建立。你可以選擇：(1) 等隔日 0:00 重置 (2) 升級進階派單會員。

**Q: 一鍵加價會不會被司機認為我「故意低價陷阱」？**  
A: 不會，因為：(1) 系統會即時提示你「初始定價過低」(2) 加價歷史對司機完全公開 (3) 低於區域均價 80% 的訂單無法加價。我們設計了多重機制保護加價單的市場信任。
