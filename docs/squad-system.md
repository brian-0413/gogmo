# 小隊互助系統規格

## 功能概述

司機可以組成小隊（最多 10 人），當接單後有狀況無法跑時，可以在小隊內轉單給隊友。轉單費用 5%（從轉出司機扣），比直接退單的 10% 便宜一半，鼓勵司機互助而不是退單。

### 商業價值

| 目標 | 機制 |
|------|------|
| 降低退單率 | 轉單 5% 比退單 10% 便宜，司機有動力找隊友接手而不是直接退 |
| 增加司機黏著度 | 小隊形成社交關係，司機不容易流失 |
| 社交裂變拉新 | 司機為了組隊會主動邀請朋友註冊，每個司機都變成推廣員 |
| 提升接單率 | 註冊人數上來後，接單池變大，派單方的單更容易被接走 |

## 小隊管理

### 基本規則

| 規則 | 設定 |
|------|------|
| 小隊人數上限 | 10 人 |
| 司機可加入小隊數 | 1 個（MVP） |
| 建立者特殊權限 | 解散小隊、邀請成員 |
| 一般成員權限 | 邀請成員、自己退出 |
| 踢人功能 | 無（建立者只能解散重編） |

### 角色定義

| 角色 | 說明 |
|------|------|
| 建立者 | 創建小隊的人，可以解散小隊和邀請成員。如果建立者退出小隊，建立者身份自動轉給最早加入的成員 |
| 成員 | 可以邀請新成員、可以自己退出、可以發起和接受轉單 |

### 小隊生命週期

```
建立小隊 → 邀請成員 → 成員加入 → 日常互助轉單
                                    ↓
                        成員自行退出 / 建立者解散重編
```

### 邀請流程

1. 任何成員都可以邀請
2. 被邀請人收到邀請通知
3. 被邀請人確認加入
4. 如果被邀請人已有小隊，提示「你已在其他小隊中，需先退出才能加入」
5. 小隊人數已達 10 人時，無法再邀請

## 轉單機制

### 轉單流程

```
司機 A 接了一張單
  ↓
A 有狀況無法跑
  ↓
A 在訂單上點「請求小隊支援」
  ↓
小隊所有成員收到通知：
  「隊友 A 的行程需要支援：4/1 14:00 板橋→桃機 $1,400」
  ↓
隊友 B 點「我可以接」
  ↓
系統檢查：
  ✓ B 的車型是否相容（大車可接小車單，小車不能接大車單）
  ✓ 是否在行程前 3 小時以上
  ↓
通知派單方：
  「訂單 #xxx 的司機要從 A 換成 B，請確認」
  ↓
派單方同意
  ↓
轉單完成：
  - 訂單的承接司機改為 B
  - 從 A 的帳戶扣 5% 轉單費
  - 紀錄轉單歷史
  ↓
派單方拒絕
  ↓
轉單失敗，訂單還是 A 的
  A 可以再發起一次支援，或直接退單（扣 10%）
```

### 轉單規則

| 規則 | 設定 |
|------|------|
| 誰可以發起 | 只有接單的司機本人 |
| 轉單費用 | 5%（從轉出的司機扣） |
| 退單費用（對比） | 10%（從退單的司機扣） |
| 轉單次數限制 | 不限次數，但只能在小隊成員之間互轉（A→B→C 都可以） |
| 時間限制 | 行程開始前 3 小時內不能轉單 |
| 車型檢查 | 接手的司機車型必須能載這筆訂單的乘客（大車可接小車單，小車不能接大車單） |
| 派單方同意 | 必須。轉單需要原派單方確認，派單方有權拒絕 |
| 轉單後負責人 | 接手的司機是新負責人。客人投訴找接手司機 |

### 時間鎖定

行程開始前 3 小時，訂單進入「鎖定」狀態：
- 不能轉單
- 不能退單（行程前 3 小時內退單費用仍為 10%）
- 司機必須履行

```
行程時間：14:00
鎖定時間：11:00 起
  - 11:00 之前：可以轉單（5%）或退單（10%）
  - 11:00 之後：不能轉單也不能退單
```

### 費用計算範例

```
訂單金額：$1,400

情境一：司機 A 直接退單
  → A 被扣 10% = $140
  → 訂單回到接單大廳重新等人接

情境二：司機 A 轉給隊友 B
  → A 被扣 5% = $70（比退單省 $70）
  → B 接手這張單，完成後 B 拿到正常收入
  → 派單方不受影響，訂單不會回到大廳

情境三：A 轉給 B，B 又轉給 C
  → A 被扣 5% = $70
  → B 被扣 5% = $70
  → C 接手完成
  → 平台多收了 $140 轉單費
```

### 派單方考量

派單方同意或拒絕時主要看：
- 接手司機的車型是否適合（系統已自動檢查，但派單方可以做最終判斷）
- 是否影響客人體驗
- 接手司機的評價/信譽（未來功能）

## 通知流程

| 事件 | 通知誰 | 內容 |
|------|--------|------|
| 司機發起支援 | 小隊所有成員 | 「隊友 X 的行程需要支援：...」 |
| 隊友回應「我可以接」 | 派單方 | 「訂單 #xxx 司機要從 A 換成 B，請確認」 |
| 派單方同意 | 原司機 A + 接手司機 B | 「轉單完成」 |
| 派單方拒絕 | 原司機 A + 接手司機 B | 「派單方拒絕轉單」 |
| 沒人回應（行程前 3 小時） | 原司機 A | 「小隊無人回應，此單即將鎖定，請儘速處理」 |

## 資料庫設計

```prisma
// 小隊
model Squad {
  id          String        @id @default(cuid())
  name        String        // 小隊名稱
  founderId   String        // 建立者 ID
  founder     Driver        @relation("SquadFounder", fields: [founderId], references: [id])
  maxMembers  Int           @default(10)
  members     SquadMember[]
  createdAt   DateTime      @default(now())
  updatedAt   DateTime      @updatedAt
}

// 小隊成員
model SquadMember {
  id        String   @id @default(cuid())
  squadId   String
  squad     Squad    @relation(fields: [squadId], references: [id])
  driverId  String   @unique // 一個司機只能在一個小隊
  driver    Driver   @relation(fields: [driverId], references: [id])
  joinedAt  DateTime @default(now())

  @@unique([squadId, driverId])
}

// 轉單紀錄
model OrderTransfer {
  id              String   @id @default(cuid())
  orderId         String
  order           Order    @relation(fields: [orderId], references: [id])
  fromDriverId    String   // 轉出司機
  fromDriver      Driver   @relation("TransferFrom", fields: [fromDriverId], references: [id])
  toDriverId      String   // 接手司機
  toDriver        Driver   @relation("TransferTo", fields: [toDriverId], references: [id])
  squadId         String   // 哪個小隊
  squad           Squad    @relation(fields: [squadId], references: [id])
  reason          String?  // 轉單原因
  transferFee     Int      // 轉單費用金額（訂單金額 * 5%）
  status          String   @default("pending")
  // pending = 等待隊友接、accepted = 隊友已接等派單方確認、
  // approved = 派單方已同意、rejected = 派單方拒絕、cancelled = 取消
  dispatcherNote  String?  // 派單方備註（拒絕原因等）
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

// 在 Order model 加入：
model Order {
  // ... 現有欄位 ...
  isLocked        Boolean  @default(false) // 行程前 3 小時鎖定
  transfers       OrderTransfer[]
}

// 在 Driver model 加入：
model Driver {
  // ... 現有欄位 ...
  squadMembership SquadMember?
  foundedSquads   Squad[]         @relation("SquadFounder")
  transfersOut    OrderTransfer[] @relation("TransferFrom")
  transfersIn     OrderTransfer[] @relation("TransferTo")
}
```

## API 端點

### 小隊管理

| Method | 端點 | 說明 |
|--------|------|------|
| POST | /api/squads | 建立小隊 |
| GET | /api/squads/my | 取得我的小隊資訊和成員列表 |
| POST | /api/squads/invite | 邀請成員（帶被邀請人的 email 或 ID） |
| POST | /api/squads/join | 接受邀請加入小隊 |
| POST | /api/squads/leave | 退出小隊 |
| DELETE | /api/squads/:id | 解散小隊（只有建立者可以） |

### 轉單

| Method | 端點 | 說明 |
|--------|------|------|
| POST | /api/orders/:id/transfer | 發起轉單請求（司機端） |
| POST | /api/orders/:id/transfer/accept | 隊友接受轉單 |
| POST | /api/orders/:id/transfer/approve | 派單方同意轉單 |
| POST | /api/orders/:id/transfer/reject | 派單方拒絕轉單 |
| GET | /api/orders/:id/transfer/history | 轉單歷史紀錄 |

## 排程任務

### 訂單鎖定（每分鐘跑一次）

```
查詢所有 status = ACCEPTED 且 行程時間 - 當前時間 <= 3小時 的訂單
→ 設定 isLocked = true
→ 如果有進行中的轉單請求，自動取消並通知相關人員
```

## 開發優先順序

1. 資料庫 schema（Squad, SquadMember, OrderTransfer）
2. 小隊 CRUD API（建立、加入、退出、解散）
3. 司機端小隊管理 UI
4. 轉單 API（發起、接受）
5. 派單方確認 API（同意、拒絕）
6. 通知機制（小隊收到支援請求、派單方收到轉單確認）
7. 司機端轉單 UI（請求支援按鈕、隊友接單介面）
8. 派單方端轉單確認 UI
9. 費用計算和扣款邏輯
10. 訂單鎖定排程
11. 轉單歷史紀錄
