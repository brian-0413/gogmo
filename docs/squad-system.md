# 小隊互助系統規格

## 功能概述

司機可以組成小隊（最多 10 人），當接單後有狀況無法跑時，可以在小隊內轉單給隊友。轉單費用 3%（從轉出司機扣，比退單 10% 便宜），鼓勵司機互助。轉單司機可附加 bonus 點數增加轉單誘因，接單司機實拿車資 + bonus 點數。

### 商業價值

| 目標 | 機制 |
|------|------|
| 降低退單率 | 轉單 3% 比退單 10% 便宜，司機有動力找隊友接手而不是直接退 |
| 增加司機黏著度 | 小隊形成社交關係，司機不容易流失 |
| 社交裂變拉新 | 司機為了組隊會主動邀請朋友註冊，每個司機都變成推廣員 |
| 提升接單率 | 註冊人數上來後，接單池變大，派單方的單更容易被接走 |

## 小隊管理

### 基本規則

| 規則 | 設定 |
|------|------|
| 小隊人數上限 | 10 人 |
| 司機可加入小隊數 | 1 個 |
| 建立者特殊權限 | 解散小隊、邀請成員、踢人 |
| 一般成員權限 | 邀請成員、自己退出 |
| 邀請流程 | 由小隊長發起邀請（輸入司機車號），對方收到通知，同意後才加入 |

### 角色定義

| 角色 | 說明 |
|------|------|
| 建立者（小隊長） | 創建小隊的人，可以解散小隊、邀請成員、踢人 |
| 成員 | 可以邀請新成員、可以自己退出、可以發起和接受轉單 |

### 費用體系

| 費用類型 | 費率 | 扣款時機 | 扣款對象 |
|----------|------|----------|----------|
| 接單費 | 5% | 司機接單成功後立即扣 | 接單司機 |
| 退單費 | 10% | 司機退單後立即扣 | 退單司機 |
| 小隊轉單費 | 3% | 有人接手成功後才扣 | 轉出司機 |
| 小隊轉單 bonus | 自訂（無上限） | 發起轉單時預扣，轉給接手司機 | 轉出司機 |

**費用說明**：
- 以上費用均為行程卡片上的原始報價金額計算
- 轉單費只在「有人成功接手」時才扣，沒人接手不扣
- bonus 點數在發起轉單時預扣（點數不足不能轉單），有人接手後轉給接手司機
- bonus 點數若沒人接手（超時鎖定/派單方撤回），退還給原司機
- 點數對應：1 點 = 1 元，接手司機可在平台使用

### 邀請流程

```
小隊長輸入司機車號 → 系統搜尋司機 → 對方收到邀請通知 → 對方點擊同意 → 加入小隊
```
- 對方若已有小隊，提示「你已在其他小隊中，需先退出才能加入」
- 小隊人數已達 10 人時，無法再邀請

## 轉單機制

### 完整流程

```
司機 A 接了一張單（報價 $800）
  ↓
A 有狀況無法跑，在行程卡片上點「請求小隊支援」
  ↓
A 填寫：轉單原因 + bonus 點數（如 50 點）→ bonus 立即從 A 帳戶扣除
  ↓
A 帳戶扣 3% = $24（此時不扣，等有人接手）
  ↓
派單方收到通知：「A 司機要轉單，請確認」
  ↓
派單方同意
  ↓
訂單進入「小隊支援池」，小隊所有成員都能看到
  ↓
隊友 B 點「我要支援」→ 先搶先贏
  ↓
系統檢查：B 的車型是否相容、B 點數是否足夠（接手後實拿車資 + bonus）
  ↓
轉單完成：
  - 訂單的承接司機改為 B
  - 從 A 的帳戶扣 3% 轉單費 = $24
  - 從 A 的帳戶 bonus 50 點轉給 B
  - B 實拿 $800 + 50 點
  ↓
派單方拒絕
  ↓
bonus 點數退還給 A，訂單仍是 A 的
  A 可再發起一次支援，或直接退單（扣 10%）
```

### 轉單規則

| 規則 | 設定 |
|------|------|
| 誰可以發起 | 只有接單的司機本人 |
| 轉單費用 | 3%（從轉出的司機扣） |
| 退單費用（對比） | 10%（從退單的司機扣） |
| bonus 點數 | 最低 10 點，無上限，由轉單司機自行設定 |
| bonus 扣款時機 | 發起時預扣（點數不足不可轉單），接手成功後轉給接手司機 |
| bonus 退還時機 | 派單方拒絕 / 超時鎖定 / 派單方撤回時退還 |
| 轉單次數限制 | 不限次數，但行程前 1 小時內不可再次發起 |
| 時間限制 | 行程前 1 小時以上才能發起轉單 |
| 車型檢查 | 接手的司機車型必須能載這筆訂單的乘客 |
| 派單方同意 | 必須。轉單需要原派單方確認 |
| 轉單後負責人 | 接手的司機是新負責人 |

### 小隊支援池

派單方核准後，訂單進入小隊支援池，等待隊友認領。

每張待搶單顯示：
- 行程時間、地點（上下車）、車資
- 原司機名稱
- bonus 點數（如有）
- 「我要支援」按鈕

搶單行為：先搶先贏，同一小隊同一訂單只能一人成功搶單。

### 時間鎖定與撤回

| 時間範圍 | 行為 |
|----------|------|
| 行程前 > 1 小時 | 可發起轉單、可退單（10%） |
| 行程前 1 小時以內 | 原司機被鎖定，不可再次發起轉單 |
| 行程前 1 小時以內 | **只有派單方**可以撤回訂單（回到普通大廳或手動指派） |

```
行進時間：14:00
1 小時鎖定：13:00 起
  - 13:00 之前：可以轉單（3%）或退單（10%）
  - 13:00 之後：
    - 原司機被鎖定，不可再次發起轉單
    - 只能等派單方處理（撤回或另找人）
```

超時兜底（沒人搶單）：
- 行進前 1 小時鎖定時，池內訂單自動退回原司機
- 原司機收到通知：「沒有隊友支援，此單即將鎖定，請聯繫派單方」
- 原司機被鎖定，不可再次發起轉單

### 費用計算範例

```
訂單報價：$1,000

情境一：司機 A 直接退單
  → A 被扣 10% = 100 點
  → 訂單回到接單大廳重新等人接

情境二：司機 A 轉給隊友 B（無 bonus）
  → A 發起轉單，B 搶單成功
  → A 被扣 3% = 30 點
  → B 實拿 $1,000

情境三：A 轉給 B（附 bonus 50 點）
  → A 發起時預扣 50 點
  → B 搶單成功
  → A 被扣 3% = 30 點
  → B 實拿 $1,000 + 50 點

情境四：A 轉給 B，B 又轉給 C
  → A 被扣 3% = 30 點
  → B 被扣 3% = 30 點（接手後又無法跑）
  → C 接手完成
  → 平台共收了 60 點轉單費
```

### 派單方考量

派單方同意或拒絕時主要看：
- 接手司機的車型是否適合
- 是否影響客人體驗
- bonus 點數是否有吸引力（系統已自動檢查）

### 派單方撤回（行程前 1 小時內）

```
行程前 1 小時內，轉單池內訂單可由派單方「撤回」
→ 訂單脫離小隊池，派單方自行處理（手動找人跑 / 聯繫司機）
→ bonus 點數退還給原司機
```

## 通知流程

| 事件 | 通知誰 | 內容 |
|------|--------|------|
| 司機發起支援 | 小隊所有成員（支援池） | 「隊友 X 的行程需要支援：...」 |
| 派單方收到請求 | 派單方 | 「訂單 #xxx 的司機要轉單，請確認」 |
| 派單方同意 | 原司機 + 小隊全員 | 「轉單核准，請隊友支援」 |
| 派單方拒絕 | 原司機 | 「派單方拒絕轉單，bonus 已退還」 |
| 有人搶單成功 | 原司機 + 接手司機 | 「轉單完成」 |
| 超時鎖定 | 原司機 + 派單方 | 「沒有隊友支援，請聯繫派單方」 |
| 派單方撤回 | 原司機 | 「派單方已撤回此轉單，bonus 已退還」 |
| 收到邀請 | 被邀請司機 | 「小隊長邀請你加入：XX 小隊」 |

## 資料庫設計

### 現有 Schema（本版新增）

```prisma
// 小隊
model Squad {
  id          String        @id @default(cuid())
  name        String        // 小隊名稱
  founderId   String       // 建立者（司機）
  founder     Driver       @relation("SquadFounder", fields: [founderId], references: [id])
  maxMembers  Int           @default(10)
  members     SquadMember[]
  transfers   OrderTransfer[]
  createdAt   DateTime     @default(now())
  updatedAt   DateTime     @updatedAt
}

// 小隊成員
model SquadMember {
  id        String   @id @default(cuid())
  squadId   String
  squad     Squad    @relation(fields: [squadId], references: [id], onDelete: Cascade)
  driverId  String   @unique // 一個司機只能在一個小隊
  driver    Driver   @relation(fields: [driverId], references: [id], onDelete: Cascade)
  joinedAt  DateTime @default(now())

  @@unique([squadId, driverId])
  @@index([squadId])
}

// 轉單紀錄
model OrderTransfer {
  id            String        @id @default(cuid())
  orderId       String
  order         Order         @relation(fields: [orderId], references: [id])
  fromDriverId  String        // 轉出司機
  fromDriver    Driver        @relation("TransferFrom", fields: [fromDriverId], references: [id])
  toDriverId    String?       // null = 還在等隊友接
  toDriver      Driver?       @relation("TransferTo", fields: [toDriverId], references: [id])
  squadId       String
  squad         Squad         @relation(fields: [squadId], references: [id])
  reason        String?
  transferFee   Int           // 轉單費用（訂單金額 * 3%）
  bonusPoints   Int           @default(0) // bonus 點數
  status        TransferStatus @default(PENDING_SQUAD)
  dispatcherNote String?
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  @@index([orderId])
  @@index([fromDriverId])
  @@index([toDriverId])
  @@index([squadId])
  @@index([status])
}

// 轉單狀態
enum TransferStatus {
  PENDING_SQUAD       // 等待小隊支援池（派單方核准後）
  PENDING_DISPATCHER  // 等待派單方確認
  APPROVED            // 已完成
  REJECTED            // 派單方拒絕
  CANCELLED           // 已取消 / 撤回
  EXPIRED             // 過期（行程鎖定）
}
```

## 管理員後台

### 費率參數調整

管理員可即時調整各項費率參數：

| 參數 | 預設值 | 說明 |
|------|--------|------|
| PLATFORM_FEE_RATE | 5% | 接單費率 |
| CANCELLATION_FEE_RATE | 10% | 退單費率 |
| TRANSFER_FEE_RATE | 3% | 小隊轉單費率 |
| MIN_BONUS_POINTS | 10 | bonus 最低點數 |

實作方式：
- 費率參數存入資料庫（PlatformConfig table）或 `src/lib/constants.ts`
- 管理員後台可即時修改
- 修改後立即生效，歷史訂單不追溯

## 排程任務

### 訂單鎖定（每分鐘跑一次）

```
查詢所有 status = ACCEPTED 且 scheduledTime - now <= 1小時 的訂單
  → 設定 isLocked = true
  → 如果有 PENDING_SQUAD 轉單，超時自動標記 EXPIRED，退還 bonus 給原司機
```

### Bonus 點數退還（超時/撤回時）

```
條件觸發：
  - 超時鎖定（scheduledTime - 1小時）
  - 派單方撤回
  - 派單方拒絕
動作：
  → 退還 bonus 點數給原司機（Transaction type: RECHARGE + description）
```
